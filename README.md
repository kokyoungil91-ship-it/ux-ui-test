1. 개발 목적 및 의도

웹 기반의 UI/UX 테스트 페이지로, 실제 서비스 제작 시 필요한 핵심 컴포넌트(헤더, 이미지 섹션, 영상 플레이어, 탭 콘텐츠, 슬라이드 콘텐츠)를 직접 설계하고 구현하는 것을 목표로 하였습니다.
사용자가 어떤 디바이스에서도 동일한 경험을 제공받을 수 있도록 반응형 레이아웃, 유지보수가 용이한 컴포넌트 구조를 중점적으로 작업하였습니다.

1) Next.js 기반 컴포넌트 구조

페이지·컴포넌트를 모듈화해 유지보수가 용이함

<Header>, <Tabs>, <Slider> 등 기능별 컴포넌트 분리

코드 재사용성 증가 및 수정 시 영향 범위 최소화

2) 반응형 최적화된 메인 레이아웃

- PC·모바일 공통 대응

- 모바일 뷰에서 시각적 안정성 확보

- 고정 픽셀을 최소화하고 유연한 레이아웃 구성


 3) 이미지 & 영상 반응형 처리

영상: IntersectionObserver로
보이는 동안 자동 재생 / 벗어나면 일시정지

불필요한 모바일 데이터 사용 절약

사용자가 콘텐츠에 집중하는 경험 제공

UX 관점에서 자동 재생 문제까지 해결
